##### Explained by LSbot

Top-Down (Memoization) Approach:
In this approach, we start with the original problem and break it down into smaller subproblems. The key is to express the solution to the complex problem in terms of solutions to these smaller subproblems.
Bottom-Up (Tabulation) Approach:
Here, we begin with the simplest subproblems and gradually build up to the original problem. We solve these subproblems in order of increasing complexity.
While they might seem similar at first glance, the main difference lies in the direction of problem-solving and how we approach the overall solution.
Let's use the Fibonacci sequence as an example to illustrate both approaches:
1. Top-Down (Memoization) Approach:
We start with the problem of finding the nth Fibonacci number. We express this in terms of smaller subproblems:
F(n) = F(n-1) + F(n-2)
We then recursively solve these smaller subproblems, storing results to avoid redundant calculations.
2. Bottom-Up (Tabulation) Approach:
We start by solving the smallest subproblems first:
F(0) = 0
F(1) = 1
Then we iteratively build up to larger values of n:
F(2) = F(1) + F(0) = 1 + 0 = 1
F(3) = F(2) + F(1) = 1 + 1 = 2
F(4) = F(3) + F(2) = 2 + 1 = 3
...and so on until we reach our target n.
The key difference is that the Top-Down approach starts with the end goal and works backwards, while the Bottom-Up approach starts with the simplest cases and works forward.
Both approaches are valid ways to solve dynamic programming problems, and the choice often depends on the specific problem and personal preference. Some developers find the Top-Down approach more intuitive, as it mirrors the way we often think about problems. Others prefer the Bottom-Up approach for its often simpler implementation and potentially better space efficiency.
For more detailed information and examples, I recommend checking out the "Intro to Dynamic Programming" chapter in the Advanced Data Structures and Algorithms book. You can find it at: https://launchschool.com/books/advanced_dsa/read/introduction_to_dynamic_programming
Remember, practice is key to mastering these concepts. Try implementing both approaches for various problems to get a feel for when each might be more appropriate or intuitive.

##### 1 Maximum treasure:

- recursion is for top down methodology, bottom up does not use it ([as taught in LS](https://launchschool.com/books/advanced_dsa/read/introduction_to_dynamic_programming#####twostrategies)).
  - Top-down = memoization, usually uses recursion.
    - Focus on the original problem: Express the solution in terms of solution to smaller subproblems
    - Recursive breakdown: Break-down into sub-probmes with a recursive function
    - Memoization for efficiency: store results in a cache so you don't repeat parts of the logic.
  - Bottom-up = tabulation, usuallly uses iteration
    - Start by solving the smallest possible subproblem, then keep on doing that...
    - ... while building off the previous calculation
    - Often uses a table (array)
    - Iterative computation: relies on loops which is cheaper than recursion.

- Top down steps:
  - define base case
  - define recursive definition: This is the line where you are calling the recursive function and doing something with the return value: like n * recursiveFunction(n) for factorials. You moight even call it a few times: `return hoppingChaos(n - 1) + hoppingChaos(n - 2)}`
  - my template:
    - define cache
    - define function
      - guard for base-case(s)
      - guard for cache.has
      - recursive case
      - return result
    -  return recursiveFunction call.
-  Bottom up (my template):
    - guard clauses
    - create a cache called dp (usually an array mirroring the input, filled with 0)
    - Use for loops to set the values in dp, building on previous answers
    - return the last element in dp
